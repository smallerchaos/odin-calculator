# odin-calculator

Project can be accessed at: https://smallerchaos.github.io/odin-calculator/

## Learnings and Reflections

This was a lot more challenging than I had thought it would be since I had to add a lot of code to deal with ways to break the calculator. I also decided to make it look a bit prettier since it's the capstone project.

I put the majority of the heavy lifting and conditionals within the function of the button click events but I'm not sure if that's best practice or not. At this stage, as long as it works, I guess?

I made it over a couple of days or even a week or two so the code is a bit messy... If I were to do this again, I would probably like to write better comments and maybe make some kind of a flow. I wonder what the best order to put functions and such in for maximum legibility. Perhaps this is something that is defined in a team or in a company rather than universally as best practice.
